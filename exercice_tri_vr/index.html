    Exercice — Tri spatial VR : Grab et Socket Interactors

    Objectifs

    Dans cet exercice, vous allez d'abord explorer les deux nouvelles interactions fondamentales du toolkit XRI — le grab et le socket — à travers un exemple minimaliste sans lien avec votre jeu. Ensuite, vous adapterez ces concepts pour créer un jeu de tri spatial fonctionnel.

    Durée approximative : 3 à 4 heures

    Prérequis : Avoir complété l'exercice de peinture VR

    Note importante : Cet exercice se développe entièrement avec le XR Interaction Simulator. Vous n'avez pas accès au casque pendant la relâche — c'est prévu et l'exercice est conçu pour fonctionner sans lui. Référez-vous aux contrôles du simulateur que vous avez déjà vus en classe.

    Partie 1 — Comprendre les concepts

    Le XR Grab Interactable

    Le XR Grab Interactable est un composant que vous ajoutez à un objet pour le rendre saisissable par un contrôleur. Quand le joueur appuie sur la gâchette en pointant vers l'objet (ou en le touchant directement), l'objet suit le mouvement du contrôleur.

    Le toolkit gère automatiquement : - La détection de la main proche ou du rayon lointain - L'attachement de l'objet au contrôleur pendant le grab - Le relâchement et la physique au lâcher

    Ce que vous devez configurer : - Le Rigidbody sur l'objet (obligatoire pour la physique) - Le Collider sur l'objet (obligatoire pour la détection) - Le composant XR Grab Interactable lui-même

    Le XR Socket Interactor

    Le XR Socket Interactor est un composant que vous ajoutez à un réceptacle — un emplacement vide qui attend de recevoir un objet grabbable. Quand un objet grabbable entre dans la zone du socket et que le joueur le relâche, l'objet se verrouille en place.

    Le socket peut être configuré pour : - Accepter tous les objets grabbables - Accepter seulement certains objets (via les Interaction Layer Masks) - Afficher un visuel fantôme de l'objet attendu

    Les Interaction Layer Masks — filtrer ce qu'un socket accepte

    En VR avec XRI, les Interaction Layer Masks jouent un rôle similaire aux Layer Masks de Physics.Raycast que vous connaissez déjà : ils permettent de filtrer quels interactors peuvent interagir avec quels interactables.

    Chaque interactor et chaque interactable a un champ Interaction Layer Mask. Une interaction n'est possible que si les deux ont au moins une couche en commun.

    Configuration dans le Project Settings : 1. Edit → Project Settings → XR Interaction Toolkit 2. Dans Interaction Layers, ajoutez vos couches personnalisées (ex: Sphere, Cube, Pyramid)

    Assigner une couche à un interactable (l'objet grabbable) : - Dans le composant XR Grab Interactable, champ Interaction Layer Mask → sélectionnez uniquement la couche correspondante

    Assigner une couche à un socket (le réceptacle) : - Dans le composant XR Socket Interactor, champ Interaction Layer Mask → sélectionnez uniquement la couche que ce socket accepte

    Résultat : un socket configuré pour Sphere refusera automatiquement un cube même si le joueur tente de l'y déposer.

    Partie 2 — Exemple de référence

    ⚠️ Cet exemple est volontairement différent de votre jeu final. L'objectif est de comprendre les patterns, pas de copier le code.

    L'exemple suivant crée une scène minimaliste : une balle qui tombe sur une table, un support qui attend de la recevoir. Quand la balle est déposée dans le support, un message s'affiche.

    Scène de l'exemple

    Une table simple avec un support cylindrique au centre. La balle est suspendue en l'air au-dessus.

    Hierarchy :
├── XR Origin (XR Rig)
├── XR Interaction Manager
├── Table (Cube, Scale 1, 0.05, 1, Position 0, 0.8, 0)
├── Balle (Sphere, Scale 0.15, Position 0, 1.5, 0)
└── Support (GameObject vide, Position 0, 0.9, 0)

    Script de réaction au dépôt — ReactionDepot.cs

    Ce script est attaché au Support. Il réagit quand un objet est déposé dans le socket.

    using UnityEngine;
using UnityEngine.XR.Interaction.Toolkit;
using UnityEngine.XR.Interaction.Toolkit.Interactors;

public class ReactionDepot : MonoBehaviour
{
    // Le socket est sur ce même GameObject
    private XRSocketInteractor socket;

    void Awake()
    {
        socket = GetComponent<XRSocketInteractor>();
    }

    void OnEnable()
    {
        // S'abonner à l'événement : un objet vient d'être déposé
        socket.selectEntered.AddListener(OnObjetDepose);

        // S'abonner à l'événement : un objet vient d'être retiré
        socket.selectExited.AddListener(OnObjetRetire);
    }

    void OnDisable()
    {
        socket.selectEntered.RemoveListener(OnObjetDepose);
        socket.selectExited.RemoveListener(OnObjetRetire);
    }

    private void OnObjetDepose(SelectEnterEventArgs args)
    {
        // args.interactableObject est l'objet qui vient d'entrer dans le socket
        GameObject objetDepose = args.interactableObject.transform.gameObject;
        Debug.Log("Objet déposé : " + objetDepose.name);
    }

    private void OnObjetRetire(SelectExitEventArgs args)
    {
        Debug.Log("Objet retiré du support");
    }
}

    Points importants :

    socket.selectEntered et socket.selectExited sont des UnityEvents — ils fonctionnent comme les événements d'Input Actions que vous connaissez déjà : on s'abonne dans OnEnable, on se désabonne dans OnDisable.

    SelectEnterEventArgs contient args.interactableObject — une référence vers l'objet qui vient d'entrer. C'est par là que vous accédez à l'objet déposé pour l'interroger ou le modifier.

    Configuration de la balle dans l'exemple

    1. Ajoutez un Rigidbody à la Balle
    2. Use Gravity : coché

    3. Is Kinematic : décoché

    4. Vérifiez que la Balle a un Sphere Collider (ajouté automatiquement)

    5. Ajoutez XR Grab Interactable à la Balle

    6. Laissez tous les paramètres par défaut pour l'exemple

    Configuration du Support dans l'exemple

    1. Ajoutez XR Socket Interactor au Support

    2. Show Interactable Hover Meshes : coché (affiche un fantôme de l'objet attendu)

    3. Ajoutez le script ReactionDepot au Support

    4. Ajoutez un Sphere Collider au Support comme zone de détection

    5. Is Trigger : coché
    6. Radius : 0.15

    Pourquoi Is Trigger ? Le socket utilise un collider trigger pour détecter quand un objet grabbable entre dans sa zone. Sans trigger, Unity traiterait ce collider comme une surface physique solide.

    Partie 3 — Votre jeu : Tri spatial

    Vous allez maintenant créer un jeu de tri spatial. Le joueur se trouve dans une salle. Des objets de trois formes différentes apparaissent devant lui. Trois réceptacles au mur correspondent chacun à une forme. L'objectif : attraper chaque objet et le déposer dans le bon réceptacle.

    Ce que vous devez produire

    • Une scène VR fonctionnelle avec le simulateur
    • Trois types d'objets grabbables, chacun associé à une interaction layer distincte
    • Trois sockets, chacun n'acceptant que la bonne forme
    • Un compteur qui détecte quand les trois réceptacles sont correctement remplis et affiche un message de victoire

    Tâche 1 — Mettre en place la scène

    Créez une nouvelle scène nommée TriSpatial dans votre projet VR existant.

    Structure recommandée de la Hierarchy :

    ├── XR Origin (XR Rig)
├── XR Interaction Manager
├── XR Interaction Simulator
├── Environnement
│   ├── Sol
│   ├── Mur arrière (avec les réceptacles)
│   └── Plafond (optionnel)
├── ZoneApparition (les objets à trier)
└── Gestionnaire (script de victoire)

    Dimensions suggérées pour l'environnement :

    Élément Position Scale
    Sol (0, 0, 0) (6, 0.1, 6)
    Mur arrière (0, 1.5, 3) (6, 3, 0.1)
    XR Origin (0, 0, -2) —

    Le joueur fait face au mur arrière où seront les réceptacles. Les objets apparaissent entre le joueur et le mur.

    Tâche 2 — Créer les Interaction Layers

    Avant de créer vos objets, définissez vos couches dans Edit → Project Settings → XR Interaction Toolkit → Interaction Layers.

    Ajoutez trois couches : donnez-leur des noms qui correspondent aux trois formes que vous choisirez.

    Conseil : Choisissez des formes disponibles dans les primitives Unity (Sphere, Cube, Cylinder ou Capsule).

    Tâche 3 — Créer les objets grabbables

    Créez trois prefabs — un par forme. Chaque prefab doit :

    • Avoir un Collider adapté à sa forme
    • Avoir un Rigidbody avec gravity activée
    • Avoir un XR Grab Interactable
    • Avoir une Interaction Layer Mask configurée sur une seule couche correspondant à sa forme
    • Avoir un matériau d'une couleur distinctive

    Placez une instance de chaque prefab dans la scène, entre le joueur et le mur.

    Positions suggérées pour les trois objets :

    Objet Position
    Forme A (-1, 1, 0)
    Forme B (0, 1, 0)
    Forme C (1, 1, 0)

    Tâche 4 — Créer les réceptacles

    Créez trois GameObjects vides contre le mur arrière — un par forme attendue.

    Positions suggérées pour les réceptacles :

    Réceptacle Position
    Réceptacle A (-1, 1.2, 2.8)
    Réceptacle B (0, 1.2, 2.8)
    Réceptacle C (1, 1.2, 2.8)

    Chaque réceptacle doit avoir :

    • Un XR Socket Interactor avec une Interaction Layer Mask configurée sur une seule couche (la couche de la forme qu'il accepte)
    • Un Sphere Collider en mode trigger (Radius : 0.2) comme zone de détection
    • Un indicateur visuel (un petit Cube ou Cylinder en enfant, semi-transparent, pour montrer où déposer l'objet)

    Tâche 5 — Script de gestion du tri

    Créez un script GestionnaireTri attaché au GameObject Gestionnaire. Ce script doit :

    • Garder une référence vers les trois XRSocketInteractor
    • S'abonner aux événements selectEntered et selectExited de chacun
    • Compter le nombre de réceptacles correctement remplis
    • Afficher "Bravo ! Tri complété." dans la console quand les trois sont remplis

    Ce que vous devez adapter de l'exemple :

    L'exemple avait un seul socket et réagissait au dépôt de n'importe quel objet. Votre script doit gérer trois sockets et détecter quand tous les trois sont simultanément occupés.

    Indice : Le socket a une propriété hasSelection (bool) qui indique s'il contient actuellement un objet. Vous pouvez l'interroger dans votre méthode de vérification.

    Indice : Vous n'avez pas besoin de vérifier que c'est le bon objet dans chaque socket — la configuration des Interaction Layer Masks s'en charge déjà. Si un socket a un objet, c'est forcément le bon.

    Tâche 6 — Tester avec le simulateur

    Lancez la scène en mode Play et vérifiez :

    • [ ] Vous pouvez attraper chaque objet avec le simulateur
    • [ ] Les objets grabbés suivent le mouvement du contrôleur simulé
    • [ ] Déposer un objet dans le bon réceptacle le verrouille en place
    • [ ] Tenter de déposer un objet dans le mauvais réceptacle ne fonctionne pas
    • [ ] Quand les trois réceptacles sont remplis, le message apparaît dans la console

    Aller plus loin (optionnel)

    Ces améliorations ne sont pas évaluées, mais elles préparent bien le sommatif.

    Minuterie

    Affichez un timer qui commence au démarrage et s'arrête quand le tri est complété. Utilisez un TextMeshPro en World Space positionné dans la scène (pas en Screen Space — ça ne fonctionne pas bien en VR).

    Score basé sur le temps

    Calculez un score en fonction du temps écoulé. Plus le tri est rapide, plus le score est élevé.

    Réapparition des objets

    Si un objet tombe hors de portée (position Y inférieure à -1), réinitialisez-le à sa position de départ.

    // Indice de structure — à adapter
void Update()
{
    if (transform.position.y < -1f)
    {
        // remettre l'objet à sa position initiale
    }
}

    Questions de réflexion

    1. Quelle est la différence entre un XR Grab Interactable et un XR Simple Interactable utilisé dans l'exercice de peinture ?
    2. Pourquoi le collider du socket doit-il être en mode trigger plutôt que solide ?
    3. Pourquoi délègue-t-on la validation de la bonne forme aux Interaction Layer Masks plutôt qu'à une vérification dans le code ?
    4. Que se passerait-il si vous oubliiez de vous désabonner des événements selectEntered dans OnDisable ?
    5. Comment faudrait-il modifier le GestionnaireTri pour permettre au joueur de recommencer la partie sans relancer la scène ?
