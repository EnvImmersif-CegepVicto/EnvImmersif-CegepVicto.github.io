    Raycasting 3D dans Unity

    Qu'est-ce qu'un raycast?

    Un raycast (lancer de rayon) est une technique qui consiste à projeter un rayon invisible dans une direction donnée pour détecter ce qu'il touche.

    Analogie du monde réel

    Imaginez que vous pointez un laser :

    • Le laser part de votre position
    • Il voyage en ligne droite dans la direction où vous pointez
    • Il s'arrête dès qu'il touche quelque chose
    • Vous pouvez alors savoir quoi a été touché et où exactement (point sur la cible)

    C'est exactement ce que fait un raycast.

    Pourquoi le raycast est-il fondamental?

    Le raycasting est la base de presque toutes les interactions en 3D :

    • Cliquer sur un objet : Lancer un rayon depuis la caméra vers la position du clic
    • Détection de sol : Vérifier si un personnage touche le sol
    • Ligne de vue : Savoir si un ennemi peut "voir" le joueur
    • Placement d'objets en AR : Trouver où placer un objet virtuel sur une surface réelle

    En réalité augmentée, nous utilisons constamment le raycasting pour savoir où l'utilisateur touche l'écran et quelle surface correspond à l'interaction.

    Anatomie d'un raycast

    Un raycast a besoin de trois infromations principales :

    1. Point de départ (Origin)

    D'où part le rayon?

    • Position de la caméra
    • Position d'un objet
    • Position du doigt sur l'écran (converti en point 3D)

    2. Direction

    Dans quelle direction va le rayon?

    • Direction de la caméra (Camera.main.transform.forward)
    • Direction vers le bas (Vector3.down)
    • Direction calculée entre deux points

    3. Distance maximale (optionnelle)

    Jusqu'où le rayon peut-il voyager avant d'abandonner?

    • Utile pour limiter la portée d'une interaction
    • Améliore les performances (moins de calculs)

    Raycasting dans Unity : La syntaxe de base

    // Créer un rayon
Ray ray = new Ray(origin, direction);

// Variable pour stocker les informations de collision
RaycastHit hit;

// Lancer le rayon et vérifier s'il touche quelque chose
if (Physics.Raycast(ray, out hit))
{
    // Le rayon a touché quelque chose!
    Debug.Log("Objet touché : " + hit.collider.gameObject.name);
    Debug.Log("Point d'impact : " + hit.point);
}

    Décomposition

    Ray ray = new Ray(origin, direction);

    • Crée un rayon avec un point de départ et une direction

    RaycastHit hit;

    • Variable qui contiendra les informations sur ce qui a été touché
    • C'est comme une "fiche de rapport" de la collision

    Physics.Raycast(ray, out hit)

    • Lance le rayon dans la scène
    • Retourne true si quelque chose est touché
    • Remplit automatiquement hit avec les détails

    Informations disponibles dans RaycastHit

    Quand un raycast touche quelque chose, l'objet RaycastHit nous donne plusieurs informations utiles :

    Propriété Type Description
    hit.point Vector3 Position exacte du point d'impact
    hit.collider Collider Le composant Collider touché
    hit.collider.gameObject GameObject L'objet complet qui a été touché
    hit.distance float Distance entre l'origine et le point d'impact
    hit.normal Vector3 Vecteur perpendiculaire à la surface touchée
    hit.transform Transform Le Transform de l'objet touché

    Exemple pratique : Détecter un clic sur un objet

    using UnityEngine;

public class RaycastClick : MonoBehaviour
{
    void Update()
    {
        // Vérifier si le bouton gauche de la souris est cliqué
        if (Input.GetMouseButtonDown(0))
        {
            // Créer un rayon depuis la caméra vers la position de la souris
            Ray ray = Camera.main.ScreenPointToRay(Input.mousePosition);
            RaycastHit hit;

            // Lancer le rayon
            if (Physics.Raycast(ray, out hit))
            {
                Debug.Log("Vous avez cliqué sur : " + hit.collider.gameObject.name);

                // Changer la couleur de l'objet touché
                Renderer renderer = hit.collider.GetComponent<Renderer>();
                if (renderer != null)
                {
                    renderer.material.color = Color.red;
                }
            }
        }
    }
}

    Points importants

    Camera.main.ScreenPointToRay(Input.mousePosition)

    • Convertit la position 2D de la souris à l'écran en rayon 3D
    • C'est la méthode la plus courante pour les clics

    Input.GetMouseButtonDown(0)

    • Détecte le moment exact où le bouton est enfoncé
    • 0 = bouton gauche, 1 = bouton droit, 2 = bouton du milieu

    Placer un objet au point d'impact

    using UnityEngine;

public class PlacerObjet : MonoBehaviour
{
    public GameObject objetAPlacer; // Assignez un prefab dans l'inspecteur

    void Update()
    {
        if (Input.GetMouseButtonDown(0))
        {
            Ray ray = Camera.main.ScreenPointToRay(Input.mousePosition);
            RaycastHit hit;

            if (Physics.Raycast(ray, out hit))
            {
                // Créer l'objet au point d'impact
                Instantiate(objetAPlacer, hit.point, Quaternion.identity);
            }
        }
    }
}

    Les quaternions : Orientation en 3D

    Qu'est-ce qu'un quaternion?

    Un quaternion est une façon mathématique de représenter une rotation dans l'espace 3D.

    Pourquoi pas simplement utiliser des angles?

    En 3D, nous avons besoin de trois rotations (X, Y, Z) - on appelle ça des angles d'Euler. Le problème avec les angles d'Euler :

    1. Gimbal Lock : Dans certaines positions, deux axes se superposent et vous "perdez" une dimension de rotation
    2. Interpolation : Animer une rotation peut donner des résultats bizarres
    3. Ambiguïté : Plusieurs combinaisons d'angles peuvent représenter la même rotation

    Les quaternions résolvent tous ces problèmes!

    Vulgarisation : Ne pas y penser en nombres

    La bonne nouvelle : Vous n'avez presque jamais besoin de créer un quaternion manuellement!

    Unity fournit des quaternions "prêts à l'emploi" :

    // Pas de rotation
Quaternion.identity

// Rotation qui regarde vers une direction
Quaternion.LookRotation(direction)

// Rotation à partir d'angles Euler (si vraiment nécessaire)
Quaternion.Euler(0, 90, 0) // Rotation de 90° sur l'axe Y

// Rotation entre deux points
Quaternion.FromToRotation(pointA, pointB)

    Quaternion.identity - Le plus utilisé

    Quaternion.identity signifie "aucune rotation" - c'est l'orientation par défaut (0°, 0°, 0°).

    Quand l'utiliser :

    • Placer un objet sans rotation spécifique
    • Réinitialiser une rotation
    • Position "neutre"
    // Créer un cube sans rotation
Instantiate(cube, position, Quaternion.identity);

    Quaternion.LookRotation - Orienter vers une direction

    Souvent, nous voulons qu'un objet "regarde" dans une direction particulière.

    // Orienter un objet vers le point d'impact
Vector3 directionVersImpact = hit.point - transform.position;
Quaternion rotation = Quaternion.LookRotation(directionVersImpact);
Instantiate(objet, transform.position, rotation);

    Aligner un objet sur une surface (utiliser la normale)

    La normale (hit.normal) est un vecteur perpendiculaire à la surface touchée. C'est crucial pour placer des objets de manière réaliste.

    if (Physics.Raycast(ray, out hit))
{
    // Rotation qui aligne l'objet perpendiculairement à la surface
    Quaternion rotation = Quaternion.FromToRotation(Vector3.up, hit.normal);

    Instantiate(objetAPlacer, hit.point, rotation);
}

    Visualisation de la normale

            ^ hit.normal (perpendiculaire)
        |
        |
    --------  Surface

    Pourquoi Vector3.up?

    • Par défaut, les objets Unity sont orientés vers le haut (axe Y positif)
    • FromToRotation calcule la rotation nécessaire pour aller de "up" vers la normale
    • Résultat : l'objet s'aligne parfaitement avec la surface

    Filtrer les collisions avec les Layers

    Parfois, nous voulons ignorer certains objets. Unity utilise des Layers (couches) pour catégoriser les objets.

    Créer et assigner des layers

    1. Sélectionnez un GameObject
    2. En haut de l'Inspector, cliquez sur "Layer" → "Add Layer..."
    3. Nommez votre layer (ex: "Ignorable")
    4. Assignez ce layer aux objets que vous voulez ignorer

    Utiliser un LayerMask dans le raycast

    // Créer un masque qui ignore la couche "Ignorable"
public LayerMask layerMask;

void Update()
{
    if (Input.GetMouseButtonDown(0))
    {
        Ray ray = Camera.main.ScreenPointToRay(Input.mousePosition);
        RaycastHit hit;

        // Raycast avec masque de layer
        if (Physics.Raycast(ray, out hit, Mathf.Infinity, layerMask))
        {
            Debug.Log("Touché : " + hit.collider.gameObject.name);
        }
    }
}

    Configuration du LayerMask dans l'Inspector

    1. Dans l'Inspector de votre script, vous verrez Layer Mask
    2. Décochez les layers que vous voulez ignorer
    3. Les objets sur ces layers seront invisibles pour ce raycast

    Astuce : Vous pouvez aussi inverser la logique en code :

    // Ignorer seulement la couche "Ignorable"
int layerMask = ~LayerMask.GetMask("Ignorable");
Physics.Raycast(ray, out hit, Mathf.Infinity, layerMask);

    Le ~ inverse le masque (ignorer au lieu d'inclure).

    Visualiser les rayons en mode Scene

    Pour déboguer, il est utile de voir les rayons :

    // Dessiner une ligne visible dans la Scene (pas en Game)
Debug.DrawRay(origin, direction * distance, Color.red, 2f);

// OU
Debug.DrawLine(startPoint, endPoint, Color.green, 2f);

    Paramètres :

    • Couleur : Color.red, Color.green, etc.
    • Durée : Temps en secondes (2f = 2 secondes)

    Ces lignes sont visibles uniquement dans la vue Scene de l'éditeur Unity.

    Bonnes pratiques

    1. Toujours vérifier si la caméra existe

    if (Camera.main == null)
{
    Debug.LogError("Aucune caméra principale trouvée!");
    return;
}

    2. Utiliser une distance maximale pour la performance

    float maxDistance = 100f;
if (Physics.Raycast(ray, out hit, maxDistance))
{
    // ...
}

    3. Vérifier les composants avant de les utiliser

    Renderer renderer = hit.collider.GetComponent<Renderer>();
if (renderer != null)
{
    renderer.material.color = Color.blue;
}
else
{
    Debug.LogWarning("Pas de Renderer sur cet objet!");
}

    4. Les objets doivent avoir un Collider

    Pour qu'un raycast détecte un objet, cet objet doit avoir un composant Collider (BoxCollider, SphereCollider, MeshCollider, etc.).

    Différences entre Raycast et RaycastAll

    Physics.Raycast()

    • S'arrête au premier objet touché
    • Plus performant
    • Parfait pour la plupart des cas

    Physics.RaycastAll()

    • Retourne tous les objets touchés le long du rayon
    • Retourne un tableau de RaycastHit
    • Utile pour voir à travers des objets, compter les collisions
    RaycastHit[] hits = Physics.RaycastAll(ray);
foreach (RaycastHit hit in hits)
{
    Debug.Log("Traversé : " + hit.collider.gameObject.name);
}

    Résumé des concepts clés

    Concept Utilité
    Ray Ligne invisible qui part d'un point dans une direction
    RaycastHit Informations sur ce qui a été touché
    hit.point Position exacte de l'impact (crucial pour placer des objets)
    hit.normal Orientation de la surface (pour aligner des objets)
    Quaternion.identity Pas de rotation (0°, 0°, 0°)
    Quaternion.FromToRotation Calculer rotation entre deux directions
    LayerMask Filtrer quels objets peuvent être touchés
