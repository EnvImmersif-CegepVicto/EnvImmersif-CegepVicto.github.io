    Exercice guidé - Peinture 3D avec Raycasting

    Objectifs de l'exercice

    Dans cet exercice, vous allez créer une petite application de "peinture 3D" qui permet de :

    1. ✅ Placer des cubes colorés en cliquant dans l'espace 3D
    2. ✅ Supprimer des cubes avec le clic droit
    3. ✅ Ignorer des objets en mouvement qui ne doivent pas interférer avec le placement
    4. ✅ Changer la couleur des cubes à placer

    Durée approximative : 45-60 minutes

    Étape 1 : Configuration de la scène

    1.1 Créer un nouveau projet Unity

    1. Ouvrez Unity Hub
    2. Créez un nouveau projet 3D
    3. Nommez-le RaycastingExercice

    1.2 Préparer la scène

    1. Créer un sol :

    2. GameObject → 3D Object → Plane

    3. Renommez-le "Sol"

    4. Scale : (5, 1, 5) pour avoir une grande surface

    5. Ajuster la caméra :

    6. Sélectionnez Main Camera

    7. Position : (0, 10, -10)
    8. Rotation : (45, 0, 0)
    9. Vous devriez voir le sol en perspective

    1.3 Créer un prefab de cube

    1. Scale : (0.5, 0.5, 0.5) pour faire un petit cube
    2. Renommez le prefab "CubePrefab"

    Étape 2 : Script de placement de base

    2.1 Créer le script principal

    1. Dans le dossier Scripts
    2. Nommez-le PlacementRaycast
    3. Double-cliquez pour l'ouvrir dans votre éditeur de code
    4. Placer un CubePrefab à l'endroit où un rayon frappe le Sol, à chaque clique.
    5. Ajouter un paramètre de couleur configurable pour la couleur du cube

    2.2 Configurer le script

    1. Créez un GameObject vide : GameObject → Create Empty
    2. Renommez-le "Utilisateur"
    3. Attachez le script PlacementRaycast au GameManager
    4. Dans l'Inspector, glissez le CubePrefab dans le champ Cube Prefab

    Testez : Lancez le jeu et cliquez sur le sol. Des cubes rouges devraient apparaître!

    Étape 3 : Ajouter la suppression de cubes

    3.1 Modifier le script pour gérer le clic droit

    1. Utilisez une nouvelle Action configurée pour détecter le clique de droite
    2. Utilisez les tag pour détecter si le GameObject touché est un cube

    Testez : Vous devriez pouvoir placer des cubes avec le clic gauche et les supprimer avec le clic droit!

    Étape 4 : Système de sélection de couleurs

    4.1 Ajouter des touches pour changer la couleur

    1. Créez l'Action et les Binding nécessaires pour détecter 5 touches différentes
    2. Programmer l'événement pour changer la couleur des cubes : rouge, vert, bleu, jaune, magenta

    Testez : Appuyez sur les touches 1-5 pour changer de couleur, puis placez des cubes!

    Étape 5 : BONIFICATION - Ajouter un objet en mouvement à ignorer

    C'est ici que nous appliquons les concepts de LayerMask.

    5.1 Créer un objet en mouvement

    1. GameObject → 3D Object → Sphere
    2. Renommez-la "SphereEnMouvement"
    3. Position : (0, 2, 0)
    4. Changez sa couleur pour la distinguer :
    5. Sélectionnez la sphère
    6. Dans l'Inspector, sous Mesh Renderer → Materials → Element 0
    7. Changez la couleur en orange ou autre couleur distinctive

    5.2 Créer un script de mouvement simple

    1. Create → C# Script → MouvementSimple
    2. Attachez ce script à la SphereEnMouvement
    3. Testez : la sphère devrait bouger de gauche à droite

    5.3 Créer un Layer "Ignorable"

    1. Sélectionnez la SphereEnMouvement
    2. En haut de l'Inspector, cliquez sur Layer → Add Layer...
    3. Dans un slot vide (par exemple User Layer 6), écrivez "Ignorable"
    4. Retournez à la SphereEnMouvement
    5. Changez son Layer pour "Ignorable"

    5.4 Modifier le script de placement pour ignorer ce layer

    Modifiez votre classe PlacementRaycast

    5.5 Configurer le LayerMask dans l'Inspector

    1. Sélectionnez le GameManager
    2. Dans l'Inspector, trouvez le champ Layer Mask
    3. Cliquez dessus pour voir la liste des layers
    4. Décochez "Ignorable"
    5. Laissez tout le reste coché (Default, etc.)

    Testez : - La sphère orange devrait bouger - Si vous cliquez dessus, rien ne se passe (elle est ignorée!) - Vous pouvez toujours placer des cubes sur le sol

    Étape 6 : Améliorations visuelles (optionnelles)

    6.1 Visualiser le rayon

    Ajoutez la visualisation du rayon.

    6.2 Indicateur de couleur actuelle

    Créez un nouveau script IndicateurCouleur.cs

    Puis :

    1. Créez un petit cube dans la scène (GameObject → 3D Object → Cube)
    2. Positionnez-le dans un coin de l'écran visible (ex: Position (3, 1, 0), Scale (0.3, 0.3, 0.3))
    3. Attachez le script IndicateurCouleur au cube
    4. Utilisez une technique au choix pour que le cube soit de la couleur actuelle de placement des cubes

    Maintenant, ce petit cube montre la couleur active!

    Étape 7 : Défis supplémentaires

    Si vous terminez en avance, essayez ces améliorations :

    Défi 1 : Ajuster la hauteur des cubes

    Modifiez le placement pour que les cubes soient légèrement au-dessus de la surface au lieu d'être parfaitement alignés :

    Vector3 positionAjustee = hit.point + hit.normal * 0.25f; // 0.25 unité au-dessus
GameObject nouveauCube = Instantiate(cubePrefab, positionAjustee, rotation);

    Défi 2 : Limiter le nombre de cubes

    Ajoutez une limite au nombre de cubes placés.

    Défi 3 : Ajouter un deuxième objet en mouvement

    Créez une autre sphère avec un mouvement vertical cette fois.

    Défi 4 : Effet de particules au placement

    Ajoutez un effet de particules au sol quand un cube est placé.

    Récapitulatif des concepts utilisés

    ✅ Raycasting : Détecter où l'utilisateur clique
    ✅ RaycastHit : Obtenir des informations sur la collision (point, normale)
    ✅ Quaternion.identity : Pas de rotation
    ✅ Quaternion.FromToRotation : Aligner avec la normale de surface
    ✅ LayerMask : Filtrer les objets que le raycast peut toucher
    ✅ Instantiate / Destroy : Créer et supprimer des objets dynamiquement
    ✅ Événement Performed : Détecter les clics
    ✅ Debug.Log / Debug.DrawRay : Déboguer et visualiser

    Questions de réflexion

    1. Que se passerait-il si vous enleviez le Collider du sol?
    2. Comment pourriez-vous modifier le code pour placer des cubes uniquement sur certaines surfaces (ex: seulement sur le sol, pas sur d'autres cubes)?
    3. Quel est l'avantage d'utiliser un LayerMask par rapport à vérifier le nom de l'objet avec if (hit.collider.gameObject.name == "...")?
