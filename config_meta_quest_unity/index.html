    Développement VR avec Meta Quest 2 et Unity

    Vue d'ensemble

    Cette semaine, nous passons de la réalité augmentée sur mobile à la réalité virtuelle sur casque autonome. Le Meta Quest 2 est un casque Android autonome — il tourne sous une version d'Android, ce qui signifie que plusieurs concepts vus précédemment (build Android, déploiement, ARM64) restent pertinents.

    L'objectif de la première heure est de configurer votre environnement de développement de A à Z : casque, Unity et outils de débogage. Tout le déploiement se fait via câble USB, ce qui est plus stable et plus simple en classe.

    Concepts clés avant de commencer

    Le Meta Quest 2 est un ordinateur Android autonome

    Contrairement à un casque PC-VR branché à un ordinateur puissant, le Quest 2 fait tourner les applications directement sur son propre processeur (Snapdragon XR2). Cela implique :

    • Les builds sont des .apk Android, comme une app mobile
    • Les performances sont limitées — optimiser est crucial
    • Le déploiement se fait via USB, directement depuis Unity

    OpenXR : le standard ouvert

    Meta a migré vers OpenXR comme standard d'industrie. C'est ce que nous utiliserons. OpenXR est à la VR ce qu'AR Foundation est à l'AR : une couche d'abstraction qui fonctionne sur plusieurs plateformes (Quest, Pico, HTC Vive, etc.), avec un seul code.

    XR Interaction Toolkit

    Unity fournit le XR Interaction Toolkit (XRI) pour gérer les interactions VR : raycasting avec les contrôleurs, saisie d'objets, UI en espace 3D. C'est l'équivalent du Input System, mais conçu spécifiquement pour les casques.

    Partie 1 : Préparer le casque

    1.1 Activer le mode développeur

    Le mode développeur est nécessaire pour installer des applications non publiées sur le Quest.

    1. Créez un compte sur developer.oculus.com et liez-le à votre compte Meta
    2. Dans l'application Meta Quest sur votre téléphone :
    3. Sélectionnez votre casque
    4. Paramètres → Mode développeur → Activez
    5. Redémarrez le casque si demandé

    Sans téléphone : Le mode développeur peut aussi être activé depuis developer.oculus.com dans un navigateur, sous la section de votre organisation.

    1.2 Installer les drivers USB sur Windows

    Pour que Windows reconnaisse le Quest en USB, installez les drivers Oculus :

    1. Téléchargez les Oculus ADB Drivers
    2. Décompressez et exécutez l'installateur
    3. Redémarrez si demandé

    1.3 Autoriser la connexion USB

    1. Connectez le casque à votre ordinateur via le câble USB-C fourni
    2. Mettez le casque — une boîte de dialogue apparaît pour autoriser le débogage USB
    3. Autorisez et cochez "Toujours faire confiance à cet ordinateur"

    Pour vérifier que la connexion fonctionne, ouvrez l'Invite de commandes (Win + R → cmd) et tapez :

    adb devices

    Vous devriez voir votre casque dans la liste avec le statut device.

    Si adb n'est pas reconnu : Téléchargez les Android Platform Tools, décompressez dans un dossier (ex: C:\adb), et ajoutez ce chemin aux variables d'environnement Windows (Système → Variables d'environnement → Path).

    Si le casque apparaît comme unauthorized : Remettez le casque, une nouvelle boîte de dialogue d'autorisation devrait apparaître.

    Partie 2 : Configurer Unity

    2.1 Vérifier le module Android

    Dans Unity Hub → Installs, vérifiez que votre Unity 6.3 a le module Android Build Support installé. Si non, cliquez sur l'icône ⚙️ à côté de la version → Add Modules.

    2.2 Créer le projet

    1. Dans Unity Hub, créez un nouveau projet
    2. Choisissez le template 3D (URP) — Universal Render Pipeline est recommandé pour la VR
    3. Nommez-le (ex: VRQuest)

    2.3 Installer les packages nécessaires

    Dans Window → Package Manager → Unity Registry, installez dans l'ordre :

    Package Remarque
    XR Plugin Management Gestion des plugins XR
    OpenXR Plugin Backend OpenXR pour Quest
    XR Interaction Toolkit Interactions VR (grab, ray, UI)

    Une fois XR Interaction Toolkit installé, restez dans le Package Manager, sélectionnez le package et ouvrez l'onglet Samples. Importez manuellement :

    • Starter Assets — prefabs préconfigurés pour les contrôleurs et la scène VR
    • XR Device Simulator — outil de simulation dans l'éditeur

    2.4 Configurer XR Plugin Management

    1. Edit → Project Settings → XR Plug-in Management
    2. Onglet PC, Mac & Linux : cochez OpenXR (pour le simulateur éditeur)
    3. Onglet Android : cochez OpenXR

    4. Sous l'onglet Android, cliquez sur OpenXR dans le menu de gauche → activez les features :

    5. Meta Quest Support ✅

    6. Oculus Touch Controller Profile ✅
    7. Hand Tracking (optionnel)

    Icône ⚠️ jaune : Si Unity affiche des avertissements, cliquez dessus — il vous proposera de corriger automatiquement les paramètres (API Level, architecture, etc.).

    2.5 Configurer les paramètres Android

    Dans Edit → Project Settings → Player → onglet Android (icône robot) :

    • Minimum API Level : Android 10 (API 29)
    • Scripting Backend : IL2CPP
    • Target Architectures : ARM64 uniquement
    • Graphics APIs : Vulkan en premier (supprimez OpenGL ES si présent)

    2.6 Adapter la qualité graphique pour le Quest 2

    Le Quest 2 n'est pas un PC de gaming. Dans Edit → Project Settings → Quality :

    • Utilisez un niveau de qualité Medium ou créez-en un dédié
    • Désactivez ou réduisez : ombres temps réel, ambient occlusion, bloom
    • Le Quest 2 vise 72 Hz (90 Hz en mode performance activé dans les paramètres du casque)

    Partie 3 : Préparer la scène VR

    3.1 Structure de scène de base

    Une scène VR diffère d'une scène 3D classique — la caméra est gérée par le casque, pas par Unity directement.

    1. Supprimez la Main Camera par défaut de la Hierarchy

    2. Dans le Project, naviguez vers Assets/Samples/XR Interaction Toolkit/3.3.1/Starter Assets/Prefabs et glissez le prefab XR Origin (XR Rig) dans la Hierarchy

    Ce prefab inclut :

    • XR Origin : le point de référence spatial de la scène
    • Camera Offset : ajuste la hauteur en fonction du mode de tracking
    • Main Camera : la caméra VR liée au casque
    • Left/Right Hand Controller : les deux contrôleurs préconfigurés

    Pourquoi ce prefab plutôt que le menu XR → XR Origin (VR) ? Le menu crée une structure vide sans contrôleurs. Le prefab Starter Assets est la méthode recommandée dans XRI 3.x — tout est déjà câblé.

    1. Clic droit dans la Hierarchy → XR → XR Interaction Manager

    Ce singleton coordonne toutes les interactions XR de la scène. Il n'est pas inclus dans le prefab et doit être ajouté séparément. Sans lui, rien ne fonctionne.

    3.2 Ajouter un environnement minimal

    Pour le premier test, ajoutez simplement :

    • GameObject → 3D Object → Plane — le sol (Scale 5, 1, 5)
    • Vérifiez qu'une Directional Light est présente dans la scène

    3.3 Tester sans casque — XR Interaction Simulator

    Unity permet de simuler le casque et les contrôleurs dans l'éditeur :

    1. Dans le Project, naviguez vers Assets/Samples/XR Interaction Toolkit/3.3.1/XR Device Simulator
    2. Glissez le prefab XR Interaction Simulator dans la Hierarchy
    3. En mode Play, utilisez la souris et le clavier pour simuler les mouvements
    Contrôle Action simulée
    Clic droit + souris Orienter la tête
    WASD Déplacer
    T / G Activer simulation main gauche / droite
    Clic gauche (main active) Gâchette

    Important : Retirez le prefab XR Interaction Simulator de la scène avant de faire un build pour le casque. Le laisser dans la scène peut causer des conflits avec les vrais contrôleurs.

    Partie 4 : Déployer sur le Quest 2

    4.1 Préparer le build

    1. Retirez le prefab XR Interaction Simulator de la Hierarchy s'il est présent
    2. File → Build Settings
    3. Cliquez sur Switch Platform → Android (le premier switch est long, ~5 min)
    4. Vérifiez que votre scène est dans la liste Scenes In Build — sinon cliquez Add Open Scenes
    5. Cochez Development Build pour avoir accès aux logs

    4.2 Déployer via USB

    Avec le casque connecté et reconnu :

    1. Dans Build Settings, cliquez sur Refresh à côté de Run Device
    2. Votre Quest apparaît dans la liste — sélectionnez-le
    3. Cliquez Build And Run

    Unity compile, installe et lance l'application directement dans le casque. Le premier build prend 10-20 minutes à cause de la compilation IL2CPP. Les suivants sont beaucoup plus rapides (~2-5 min) grâce au cache.

    Conseil : Pendant que ça compile, préparez votre casque et lisez les sections suivantes.

    4.3 Consulter les logs depuis Unity

    Avec le casque connecté en USB et l'app qui tourne, ouvrez dans Unity :

    Window → Analysis → Android Logcat

    Filtrez sur Unity pour ne voir que les messages de votre application. C'est ici que vos Debug.Log() apparaissent et que les erreurs sont visibles.

    Si Android Logcat n'apparaît pas : Installez le package Android Logcat depuis le Package Manager.

    Partie 5 : Dépannage courant

    Le casque n'apparaît pas dans Build Settings

    • Vérifiez la connexion USB et que le mode développeur est activé
    • Mettez le casque — une boîte de dialogue d'autorisation attend peut-être votre réponse
    • Essayez un autre port USB ou un autre câble (les câbles charge-only ne fonctionnent pas)
    • Réinstallez les Oculus ADB Drivers

    L'application plante immédiatement au démarrage

    Les causes les plus fréquentes :

    • Architecture non supportée → vérifiez ARM64 uniquement dans Player Settings
    • API Level trop bas → minimum API 29
    • OpenXR non configuré sur l'onglet Android de XR Plug-in Management

    L'écran reste noir dans le casque

    • XR Origin absent de la scène
    • La caméra est à l'intérieur d'un objet opaque
    • OpenXR ou Meta Quest Support non activés

    Les contrôleurs ne bougent pas / ne répondent pas

    • Oculus Touch Controller Profile non activé dans les features OpenXR
    • XR Interaction Manager absent de la scène
    • Vérifiez que les GameObjects des contrôleurs ont bien le composant XR Controller

    Résumé — Checklist de configuration

    Casque :

    • [ ] Compte développeur Meta créé
    • [ ] Mode développeur activé
    • [ ] Drivers Oculus installés sur Windows
    • [ ] Connexion USB autorisée dans le casque
    • [ ] Casque visible dans l'Invite de commandes avec adb devices

    Unity :

    • [ ] Module Android Build Support présent dans Unity 6.3
    • [ ] Packages installés : XR Plugin Management, OpenXR, XR Interaction Toolkit
    • [ ] OpenXR activé sur les onglets PC et Android
    • [ ] Meta Quest Support + Oculus Touch Controller Profile activés
    • [ ] Minimum API 29, IL2CPP, ARM64 uniquement, Vulkan

    Scène :

    • [ ] Main Camera par défaut supprimée
    • [ ] Prefab XR Origin (XR Rig) (Starter Assets) présent
    • [ ] XR Interaction Manager présent (ajouté via menu XR)
    • [ ] Sol et éclairage de base présents
    • [ ] XR Interaction Simulator retiré avant le build

    Déploiement :

    • [ ] Plateforme Android sélectionnée (Switch Platform)
    • [ ] Scène ajoutée dans Build Settings
    • [ ] Quest visible dans Run Device
    • [ ] Premier build réussi — application visible dans le casque
