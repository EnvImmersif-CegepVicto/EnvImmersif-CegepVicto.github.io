    Exercice — Feedback multimodal : haptics et audio spatial

    Objectifs

    Dans cet exercice, vous allez ajouter une couche de feedback sensoriel à votre jeu de tri spatial. Un jeu qui fonctionne devient un jeu qui se sent bien.

    Concrètement, vous allez :

    1. ✅ Déclencher des vibrations haptiques sur le grab et le dépôt
    2. ✅ Jouer un son positionnel sur chaque interaction clé
    3. ✅ Comprendre la différence entre AudioSource en World Space et en Screen Space en VR

    Durée approximative : 2 heures

    Prérequis : Avoir complété l'exercice de tri spatial (TriSpatial)

    Cette séance se fait avec le casque. Profitez-en — le feedback haptique ne peut pas être évalué sans le sentir.

    Partie 1 — Comprendre les concepts

    Les haptics en XRI

    En XRI 3.x, les vibrations haptiques sont envoyées via XRBaseController. La méthode centrale est :

    controller.SendHapticImpulse(amplitude, duration);
    • amplitude : intensité de la vibration, de 0.0f (rien) à 1.0f (maximum)
    • duration : durée en secondes

    Pour accéder au contrôleur depuis un script, on peut interroger l'interactor qui a effectué l'action. Les événements de XRGrabInteractable et XRSocketInteractor fournissent une référence vers l'interactor dans leurs EventArgs.

    L'audio spatial en VR

    En VR, le son doit se comporter comme dans le monde réel — il doit venir d'une direction et s'atténuer avec la distance.

    Unity gère cela via le composant AudioSource combiné à un AudioListener. En VR, l'AudioListener est automatiquement placé sur la caméra principale (dans le XR Origin), ce qui reproduit la perception auditive naturelle.

    La clé est le champ Spatial Blend de l'AudioSource :

    Valeur Comportement
    0 (2D) Son identique dans les deux oreilles, peu importe la position — à éviter en VR sauf pour la musique d'ambiance
    1 (3D) Son positionnel, atténué par la distance et la direction — à utiliser pour les interactions

    Pour un son positionnel sur une interaction, l'AudioSource doit être sur l'objet lui-même (ou à sa position), pas sur la caméra.

    Lier feedback et événements XRI

    Les composants XRGrabInteractable et XRSocketInteractor exposent des UnityEvents que vous connaissez déjà depuis l'exercice de tri. Ces mêmes événements transportent les EventArgs qui donnent accès à l'interactor — et donc au contrôleur pour les haptics.

    selectEntered → args.interactorObject → GetComponent<XRBaseController>() → SendHapticImpulse()

    Partie 2 — Exemple de référence

    Vous repartez de la scène minimale de l'exercice précédent — la table, la balle et le support. Ajoutez le feedback sur cette scène avant de toucher à votre jeu de tri.

    L'exemple illustre les deux patterns : haptics au grab de la balle, son positionnel au verrouillage dans le support.

    Script de feedback sur un objet grabbable — FeedbackGrab.cs

    Ce script est attaché à la balle.

    using UnityEngine;
using UnityEngine.XR.Interaction.Toolkit;
using UnityEngine.XR.Interaction.Toolkit.Interactables;
using UnityEngine.XR.Interaction.Toolkit.Interactors;

[RequireComponent(typeof(XRGrabInteractable))]
[RequireComponent(typeof(AudioSource))]
public class FeedbackGrab : MonoBehaviour
{
    [Header("Haptique")]
    [SerializeField] private float amplitudeGrab = 0.5f;
    [SerializeField] private float dureeGrab = 0.1f;

    [Header("Audio")]
    [SerializeField] private AudioClip sonGrab;

    private XRGrabInteractable grabInteractable;
    private AudioSource audioSource;

    void Awake()
    {
        grabInteractable = GetComponent<XRGrabInteractable>();
        audioSource = GetComponent<AudioSource>();

        // Configurer l'AudioSource pour du son positionnel
        audioSource.spatialBlend = 1f; // 100% 3D
        audioSource.rolloffMode = AudioRolloffMode.Logarithmic;
        audioSource.maxDistance = 5f;
    }

    void OnEnable()
    {
        grabInteractable.selectEntered.AddListener(OnGrabEntered);
        grabInteractable.selectExited.AddListener(OnGrabExited);
    }

    void OnDisable()
    {
        grabInteractable.selectEntered.RemoveListener(OnGrabEntered);
        grabInteractable.selectExited.RemoveListener(OnGrabExited);
    }

    private void OnGrabEntered(SelectEnterEventArgs args)
    {
        // Récupérer le contrôleur depuis l'interactor
        var controller = args.interactorObject.transform.GetComponent<XRBaseController>();

        controller.SendHapticImpulse(amplitudeGrab, dureeGrab);

        // Jouer le son à la position de l'objet
        audioSource.PlayOneShot(sonGrab);
    }

    private void OnGrabExited(SelectExitEventArgs args)
    {
        // Vibration plus courte et moins forte au relâchement
        var controller = args.interactorObject.transform.GetComponent<XRBaseController>();

        controller.SendHapticImpulse(amplitudeGrab * 0.3f, dureeGrab * 0.5f);
    }
}

    Script de feedback sur un socket — FeedbackSocket.cs

    Ce script est attaché au support.

    using UnityEngine;
using UnityEngine.XR.Interaction.Toolkit;
using UnityEngine.XR.Interaction.Toolkit.Interactors;

[RequireComponent(typeof(XRSocketInteractor))]
[RequireComponent(typeof(AudioSource))]
public class FeedbackSocket : MonoBehaviour
{
    [Header("Haptique")]
    [SerializeField] private float amplitudeDepot = 0.8f;
    [SerializeField] private float dureeDepot = 0.2f;

    [Header("Audio")]
    [SerializeField] private AudioClip sonDepot;

    private XRSocketInteractor socket;
    private AudioSource audioSource;

    void Awake()
    {
        socket = GetComponent<XRSocketInteractor>();
        audioSource = GetComponent<AudioSource>();

        audioSource.spatialBlend = 1f;
        audioSource.maxDistance = 5f;
    }

    void OnEnable()
    {
        socket.selectEntered.AddListener(OnDepotEntered);
    }

    void OnDisable()
    {
        socket.selectEntered.RemoveListener(OnDepotEntered);
    }

    private void OnDepotEntered(SelectEnterEventArgs args)
    {
        // Récupérer le contrôleur qui vient de lâcher la balle
        var controller = args.interactorObject.transform.GetComponent<XRBaseController>();

        controller.SendHapticImpulse(amplitudeDepot, dureeDepot);

        audioSource.PlayOneShot(sonDepot);
    }
}

    Points importants :

    args.interactorObject dans le contexte d'un socket correspond au Near-Far Interactor du contrôleur — on remonte jusqu'au XRBaseController sur le même GameObject pour envoyer la vibration.

    audioSource.PlayOneShot(clip) joue le son sans interrompre un son déjà en cours — préférable à audioSource.Play() pour les sons d'interaction qui peuvent se superposer rapidement.

    spatialBlend = 1f sur l'AudioSource est la ligne la plus importante pour l'immersion. Sans elle, le son est identique dans les deux oreilles et perd tout ancrage spatial.

    Partie 3 — Votre jeu : ajouter le feedback

    Ouvrez votre scène TriSpatial. Vous allez enrichir chaque objet grabbable et chaque réceptacle avec du feedback sensoriel.

    Obtenir des sons

    Vous avez besoin d'au minimum deux sons courts : un pour le grab, un pour le dépôt dans un socket. Quelques sources gratuites compatibles avec les projets scolaires :

    • freesound.org — licence CC, vérifiez les attributions requises
    • kenney.nl/assets — entièrement libre de droits

    Privilégiez des fichiers .wav ou .ogg, courts (moins de 1 seconde pour les sons d'interaction).

    Tâche 1 — Feedback sur le grab

    Adaptez le script FeedbackGrab de l'exemple pour l'appliquer à vos objets grabbables.

    Ce que vous devez adapter : - Le script de l'exemple a un seul AudioClip. Votre adaptation doit permettre d'assigner des sons différents selon la forme de l'objet (un son doux pour la sphère, un son mat pour le cube, etc.) — ou vous pouvez utiliser le même son pour tous si vous jugez que c'est le bon choix de design. - L'amplitude et la durée doivent être configurables depuis l'Inspector pour chaque objet, pas codées en dur.

    Vérifiez sur le casque : - [ ] Attraper un objet déclenche une vibration et un son - [ ] Le son vient bien de la direction de l'objet (tournez la tête pour valider) - [ ] Relâcher l'objet dans le vide déclenche une vibration plus faible

    Tâche 2 — Feedback sur le dépôt correct

    Adaptez le script FeedbackSocket pour vos réceptacles.

    Ce que vous devez adapter : - L'amplitude du dépôt doit être plus forte que celle du grab — le dépôt correct est le moment de récompense principal du jeu. - Le son du dépôt doit être différent du son du grab, même si vous n'avez qu'un seul fichier audio par défaut.

    Vérifiez sur le casque : - [ ] Déposer un objet dans le bon réceptacle déclenche une vibration distincte et un son - [ ] Le son de dépôt vient de la position du réceptacle

    Tâche 3 — Feedback de victoire

    Votre GestionnaireTri détecte déjà quand les trois réceptacles sont remplis. Ajoutez un feedback de victoire :

    • Une vibration sur les deux contrôleurs simultanément, plus longue que les vibrations d'interaction
    • Un son d'ambiance ou de victoire joué en 2D (Spatial Blend à 0) — ce son doit envelopper le joueur, pas venir d'une direction précise

    Pour accéder aux deux contrôleurs depuis le GestionnaireTri, vous aurez besoin de les référencer. Une façon simple :

    // Indice de structure — à adapter
[SerializeField] private XRBaseController controleurGauche;
[SerializeField] private XRBaseController controleurDroit;

private void DeclencherVictoireHaptique()
{
    // À vous de compléter
}

    Indice : Les contrôleurs se trouvent dans la hiérarchie du XR Origin. Vous pouvez les assigner directement dans l'Inspector plutôt que de les chercher par code.

    Tâche 4 — Comparer avec et sans feedback

    Avant de passer à autre chose, faites le test suivant :

    1. Désactivez temporairement tous vos scripts de feedback (décochez-les dans l'Inspector)
    2. Jouez une partie complète
    3. Réactivez les scripts et rejouez

    Questions de réflexion

    1. Pourquoi configure-t-on spatialBlend = 1f par code dans Awake() plutôt que de le faire directement dans l'Inspector ?
    2. Quelle est la différence entre audioSource.Play() et audioSource.PlayOneShot() ? Dans quel cas l'un est-il préférable à l'autre ?
    3. La vibration de victoire est plus longue que les vibrations d'interaction. Quel principe de design justifie ce choix ?
    4. Si un joueur joue sans casque audio (seulement avec les hautparleurs du Quest), l'audio spatial fonctionne-t-il quand même ? Pourquoi ?
    5. Comment faudrait-il modifier FeedbackGrab pour que l'amplitude de la vibration soit proportionnelle à la vitesse à laquelle le joueur attrape l'objet ?
