    Exercice pratique - Passage du raycasting 3D à la réalité augmentée

    Objectifs de l'exercice

    Dans cet exercice, vous allez adapter votre projet de peinture 3D de la semaine 1 pour qu'il fonctionne en réalité augmentée. Concrètement, vous allez :

    1. ✅ Configurer votre projet existant pour l'AR
    2. ✅ Remplacer le sol fixe par des plans détectés dynamiquement
    3. ✅ Adapter le raycasting pour toucher les plans AR
    4. ✅ Tester en simulation AR
    5. ✅ BONUS : Ajouter un système de feedback visuel pendant la détection

    Durée approximative : 60-90 minutes

    Prérequis : Avoir complété l'exercice de raycasting 3D de la semaine 1

    Étape 1 : Préparation du projet

    1.1 : Ouvrir votre projet de la semaine 1

    1. Ouvrez Unity Hub
    2. Ouvrez votre projet RaycastingExercice de la semaine 1

    3. Créez une copie de votre scène actuelle :

    4. Clic droit sur votre scène → Duplicate

    5. Renommez la copie "SceneAR"
    6. Ouvrez cette nouvelle scène

    Pourquoi une copie ? Pour garder votre version 3D classique intacte au cas où.

    1.2 : Installer les packages AR

    Suivez la Partie 1 du document "Configuration AR Foundation et Simulation AR" :

    • [ ] Installer AR Foundation
    • [ ] Installer Google ARCore XR Plugin
    • [ ] Installer XR Simulation

    1.3 : Configurer les paramètres du projet

    Suivez la Partie 2 du même document :

    • [ ] Activer XR Simulation (Desktop)
    • [ ] Activer Google ARCore (Android)
    • [ ] Configurer les paramètres Android

    Étape 2 : Transformer la scène en scène AR

    2.1 : Supprimer les éléments non compatibles

    Dans votre scène actuelle, supprimez :

    1. ❌ Le Plane (sol) - il sera remplacé par la détection de plans AR
    2. ❌ La Main Camera par défaut

    Gardez :

    • ✅ Votre GameObject "Utilisateur" (ou "GameManager") avec le script de placement
    • ✅ Le prefab de cube
    • ✅ La sphère en mouvement (si vous l'avez créée)

    2.2 : Ajouter les composants AR

    Suivez la Partie 3 du document de configuration :

    1. Ajouter XR Origin :

    2. Clic droit dans Hierarchy → XR → XR Origin (Mobile AR)

    3. Ajouter AR Session :

    4. Clic droit dans Hierarchy → XR → AR Session

    5. Configurer XR Origin :

    6. Sélectionnez XR Origin

    7. Add Component → AR Plane Manager
    8. Add Component → AR Raycast Manager

    2.3 : Créer et assigner le prefab de plan

    Suivez la Partie 6 du document de configuration pour créer un prefab de plan visualisable.

    Configuration recommandée pour cet exercice :

    • Matériau : Vert semi-transparent (Alpha ~0.4)
    • Cela permet de voir clairement où sont les surfaces détectées

    Assignez le prefab dans AR Plane Manager → Plane Prefab

    2.4 : Configurer la simulation

    1. Edit → Project Settings → XR Simulation
    2. Sélectionnez "Simulated Room" comme environnement
    3. Testez en appuyant sur Play - vous devriez voir l'environnement de simulation

    Étape 3 : Adapter le script de placement

    C'est l'étape clé ! Vous allez modifier votre script PlacementRaycast.cs pour utiliser le raycasting AR.

    3.1 : Ajouter les namespaces nécessaires

    En haut de votre script, ajoutez :

    using UnityEngine.XR.ARFoundation;
using UnityEngine.XR.ARSubsystems;

    3.2 : Ajouter une référence à AR Raycast Manager

    Dans votre classe, ajoutez :

    [SerializeField] private ARRaycastManager arRaycastManager;

    Dans Unity :

    1. Sélectionnez votre GameObject "Utilisateur"
    2. Dans l'Inspector, trouvez votre script
    3. Glissez XR Origin dans le champ AR Raycast Manager (XR Origin a le composant AR Raycast Manager)

    3.3 : Comprendre la différence entre les deux types de raycast

    Raycasting 3D classique :

    // Touche des COLLIDERS Unity
Ray ray = Camera.main.ScreenPointToRay(touchPosition);
if (Physics.Raycast(ray, out RaycastHit hit))
{
    // hit contient des infos sur le collider touché
}

    Raycasting AR (Nouvelle méthode) :

    // Touche des PLANS DÉTECTÉS par ARCore
List<ARRaycastHit> hits = new List<ARRaycastHit>();
if (arRaycastManager.Raycast(touchPosition, hits, TrackableType.PlaneWithinPolygon))
{
    // hits[0] contient des infos sur le plan AR touché
}

    3.4 : Modifier la méthode de placement

    Remplacez votre méthode de placement existante par cette version AR :

    private void OnTap(InputAction.CallbackContext context)
{
    // Obtenir la position du touch/clic
    Vector2 touchPosition = controles.Player.Point.ReadValue<Vector2>();

    // Liste pour stocker les résultats du raycast AR
    List<ARRaycastHit> hits = new List<ARRaycastHit>();

    // Lancer un raycast AR vers les plans détectés
    if (arRaycastManager.Raycast(touchPosition, hits, TrackableType.PlaneWithinPolygon))
    {
        // Prendre le premier plan touché
        Pose hitPose = hits[0].pose;

        // Créer le cube à la position du plan
        GameObject nouveauCube = Instantiate(cubePrefab, hitPose.position, Quaternion.identity);

        // Appliquer la couleur actuelle
        Renderer renderer = nouveauCube.GetComponent<Renderer>();
        if (renderer != null)
        {
            renderer.material.color = couleurActuelle;
        }

        // Assigner le tag pour pouvoir supprimer plus tard
        nouveauCube.tag = "Cube";
    }
}

    3.5 : Points importants à noter

    TrackableType.PlaneWithinPolygon :

    • Ne touche que les plans détectés (pas les objets virtuels)
    • Équivalent AR du LayerMask pour ignorer certains objets

    hits[0].pose :

    • Un Pose contient position ET rotation
    • pose.position = Vector3 de la position
    • pose.rotation = Quaternion de la rotation

    Pourquoi Quaternion.identity ?

    • Pour cet exercice, on garde les cubes non-rotés pour simplifier
    • On pourrait utiliser hitPose.rotation pour aligner avec la surface

    3.6 : Adapter la suppression

    Modifiez le système de suppression au clic droit de la même manière :

    private void OnDelete(InputAction.CallbackContext context)
{
    Vector2 touchPosition = controles.Player.Point.ReadValue<Vector2>();

    // On peut utiliser le raycast CLASSIQUE pour toucher les cubes
    Ray ray = Camera.main.ScreenPointToRay(touchPosition);
    if (Physics.Raycast(ray, out RaycastHit hit))
    {
        if (hit.collider.CompareTag("Cube"))
        {
            Destroy(hit.collider.gameObject);
        }
    }
}

    Pourquoi mélanger les deux types ?

    • AR Raycast pour toucher les plans réels détectés
    • Physics Raycast classique pour toucher les cubes virtuels (qui ont des colliders)

    Étape 5 : Amélioration - Feedback visuel

    5.1 : Ajouter un indicateur de "prêt"

    Créez un nouveau script ARStatusFeedback.cs :

    using UnityEngine;
using UnityEngine.UI;
using UnityEngine.XR.ARFoundation;

public class ARStatusFeedback : MonoBehaviour
{
    [SerializeField] private ARPlaneManager planeManager;
    [SerializeField] private Text statusText; // Nécessite un UI Text

    private int planesCount = 0;

    void Update()
    {
        // Compter les plans détectés
        planesCount = planeManager.trackables.count;

        if (planesCount == 0)
        {
            statusText.text = "Recherche de surfaces...";
            statusText.color = Color.yellow;
        }
        else
        {
            statusText.text = $"Prêt ! {planesCount} surface(s) détectée(s)";
            statusText.color = Color.green;
        }
    }
}

    5.2 : Créer l'UI

    1. Clic droit dans Hierarchy → UI → Text - TextMeshPro (Si demandé, importez TMP Essentials)
    2. Renommez le Text en "StatusText"

    3. Configurez le texte :

    4. Font Size : 24

    5. Alignment : Center

    6. Position en haut de l'écran

    7. Créez un GameObject vide "ARManager"

    8. Attachez le script ARStatusFeedback

    9. Assignez les références :

    10. AR Plane Manager (depuis XR Origin)

    11. Status Text

    Testez : Le texte devrait indiquer combien de surfaces sont détectées.

    Étape 6 : Gérer les cubes sur différents types de plans

    6.1 : Détecter le type de plan

    Modifiez votre méthode de placement pour réagir au type de plan :

    private void OnTap(InputAction.CallbackContext context)
{
    Vector2 touchPosition = controles.Player.Point.ReadValue<Vector2>();
    List<ARRaycastHit> hits = new List<ARRaycastHit>();

    if (arRaycastManager.Raycast(touchPosition, hits, TrackableType.PlaneWithinPolygon))
    {
        Pose hitPose = hits[0].pose;

        // Obtenir le plan touché
        ARPlane plane = hits[0].trackable as ARPlane;

        // Adapter selon le type de plan
        Vector3 position = hitPose.position;

        if (plane.alignment == PlaneAlignment.HorizontalUp)
        {
            // Sol ou table - position normale
            position += Vector3.up * 0.25f; // Légèrement au-dessus
        }
        else if (plane.alignment == PlaneAlignment.Vertical)
        {
            // Mur - peut-être coller au mur ?
            position += plane.normal * 0.1f; // Légèrement devant le mur
        }

        GameObject nouveauCube = Instantiate(cubePrefab, position, Quaternion.identity);

        // Appliquer couleur
        Renderer renderer = nouveauCube.GetComponent<Renderer>();
        if (renderer != null)
        {
            renderer.material.color = couleurActuelle;
        }

        nouveauCube.tag = "Cube";

        Debug.Log($"Cube placé sur : {plane.alignment}");
    }
}

    6.2 : Configuration pour tester

    Dans AR Plane Manager :

    • Changez Detection Mode à Everything

    Cela permet de détecter les murs en plus du sol.

    Récapitulatif des concepts utilisés

    ✅ AR Foundation : Framework unifié pour AR
    ✅ XR Origin : Système de coordonnées AR
    ✅ AR Session : Gestion de la session AR
    ✅ AR Plane Manager : Détection de plans
    ✅ AR Raycast Manager : Raycasting sur plans AR
    ✅ TrackableType : Filtrer ce que le raycast touche
    ✅ Pose : Position + rotation en AR
    ✅ PlaneAlignment : Type de plan (horizontal/vertical)

    Différences clés avec le raycasting 3D

    Aspect Raycasting 3D Raycasting AR
    Cible Colliders Unity Plans détectés par ARCore
    Manager Physics ARRaycastManager
    Hit type RaycastHit ARRaycastHit
    Position hit.point hit.pose.position

    Dépannage

    ❌ "ARRaycastManager is null"

    Cause : Référence non assignée

    Solution :

    1. Vérifiez que XR Origin a bien le composant AR Raycast Manager
    2. Glissez XR Origin dans le champ du script dans l'Inspector

    ❌ Le raycast ne touche jamais rien

    Vérifications :

    1. Les plans sont bien détectés (grilles visibles) ?
    2. AR Raycast Manager est sur XR Origin ?
    3. Vous utilisez bien TrackableType.PlaneWithinPolygon ?

    ❌ Les cubes apparaissent à des positions étranges

    Cause possible : Confusion entre coordinate spaces

    Solution : Utilisez toujours hits[0].pose.position directement, sans transformation

    ❌ "Namespace XR.ARFoundation could not be found"

    Cause : Packages AR non installés

    Solution :

    1. Vérifiez Package Manager → AR Foundation installé
    2. Redémarrez Unity si nécessaire
