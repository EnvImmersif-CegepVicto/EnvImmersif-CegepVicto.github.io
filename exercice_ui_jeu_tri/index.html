    Exercice — UI en espace 3D et structure de jeu

    Objectifs

    Dans cet exercice, vous allez d'abord construire une boucle de jeu complète sur un exemple minimaliste — une table et une sphère. Ensuite, vous adapterez ces concepts pour transformer votre jeu de tri en une expérience jouable de bout en bout.

    Concrètement, vous allez :

    1. ✅ Configurer un Canvas World Space avec interaction contrôleur
    2. ✅ Implémenter un Game Manager avec états (menu → jeu → fin)
    3. ✅ Afficher un timer et un score dans un HUD
    4. ✅ Adapter la structure complète à votre jeu de tri existant

    Durée approximative : 2 heures 15

    Prérequis : Avoir complété l'exercice de feedback multimodal (TriSpatial avec haptics et audio)

    Partie 1 — Exemple de référence : la sphère dans le support

    Vous repartez de la scène minimale utilisée à la séance précédente — la table, la sphère grabbable et le support (socket). L'objectif est simple : attraper la sphère et la déposer dans le support. Le jeu démarre sur un menu et se termine sur un écran de fin avec un score basé sur le temps.

    Travaillez dans cette scène avant de toucher à votre jeu de tri.

    1.1 Préparer l'Event System

    Vérifiez que votre scène contient un Event System (GameObject → UI → Event System si absent).

    1. Sélectionnez l'Event System
    2. Retirez le composant Standalone Input Module (clic droit → Remove Component)
    3. Ajoutez XR UI Input Module (Add Component → XR → XR UI Input Module)

    Sans ce remplacement, les boutons ne répondront pas au rayon du contrôleur.

    1.2 Créer les trois Canvas

    Créez trois Canvas (GameObject → UI → Canvas). Pour chacun, configurez :

    • Render Mode : World Space
    • Width : 400 / Height : 300
    • Scale : (0.005, 0.005, 0.005)
    • Retirez Graphic Raycaster → ajoutez Tracked Device Graphic Raycaster
    Canvas Nom Position
    Menu principal CanvasMenu (0, 1.8, -1) face au joueur
    Écran de fin CanvasGameOver (0, 1.8, -1) même position
    HUD CanvasHUD Enfant de la caméra (voir ci-dessous)

    Pour le CanvasHUD : glissez-le comme enfant de XR Origin → Camera Offset → Main Camera dans la Hierarchy, puis configurez sa position locale :

    Position : (0, -0.3, 1.0)
Scale : (0.001, 0.001, 0.001)

    Le HUD suit la tête du joueur. Il n'a pas besoin de Tracked Device Graphic Raycaster — le joueur ne clique pas dessus.

    1.3 Contenu des Canvas

    CanvasMenu — ajoutez à l'intérieur : - Text - TextMeshPro : titre (ex : "Attrape la sphère") - Button - TextMeshPro : texte "Jouer"

    CanvasHUD — ajoutez à l'intérieur : - Text - TextMeshPro nommé TexteTimer : texte initial "00:00"

    CanvasGameOver — ajoutez à l'intérieur : - Text - TextMeshPro : "Terminé !" - Text - TextMeshPro nommé TexteScoreFinal : texte initial "Score : 0" - Button - TextMeshPro : texte "Rejouer"

    1.4 Le Game Manager

    Créez un script GameManager sur un GameObject vide nommé GameManager.

    using UnityEngine;
using TMPro;

public class GameManager : MonoBehaviour
{
    public enum EtatJeu { Menu, EnJeu, GameOver }

    [Header("Canvas")]
    [SerializeField] private GameObject canvasMenu;
    [SerializeField] private GameObject canvasHUD;
    [SerializeField] private GameObject canvasGameOver;

    [Header("Textes")]
    [SerializeField] private TextMeshProUGUI texteTimer;
    [SerializeField] private TextMeshProUGUI texteScoreFinal;

    private EtatJeu etatActuel;
    private float tempsEcoule;
    private bool timerActif;

    void Start()
    {
        ChangerEtat(EtatJeu.Menu);
    }

    void Update()
    {
        if (timerActif)
        {
            tempsEcoule += Time.deltaTime;
            AfficherTimer();
        }
    }

    public void ChangerEtat(EtatJeu nouvelEtat)
    {
        etatActuel = nouvelEtat;
        canvasMenu.SetActive(etatActuel == EtatJeu.Menu);
        canvasHUD.SetActive(etatActuel == EtatJeu.EnJeu);
        canvasGameOver.SetActive(etatActuel == EtatJeu.GameOver);
    }

    public void CommencerJeu()
    {
        tempsEcoule = 0f;
        timerActif = true;
        AfficherTimer();
        ChangerEtat(EtatJeu.EnJeu);
    }

    public void TerminerJeu()
    {
        timerActif = false;
        int score = Mathf.Max(100, 1000 - Mathf.FloorToInt(tempsEcoule) * 10);
        texteScoreFinal.text = $"Score : {score}";
        ChangerEtat(EtatJeu.GameOver);
    }

    public void Rejouer()
    {
        UnityEngine.SceneManagement.SceneManager.LoadScene(
            UnityEngine.SceneManagement.SceneManager.GetActiveScene().name
        );
    }

    private void AfficherTimer()
    {
        int minutes = Mathf.FloorToInt(tempsEcoule / 60f);
        int secondes = Mathf.FloorToInt(tempsEcoule % 60f);
        texteTimer.text = $"{minutes:00}:{secondes:00}";
    }
}

    Assignez toutes les références dans l'Inspector du GameManager.

    1.5 Connecter les boutons

    Les boutons appellent les méthodes du GameManager via On Click () dans l'Inspector — aucun script sur les boutons.

    • Bouton "Jouer" → GameManager.CommencerJeu()
    • Bouton "Rejouer" → GameManager.Rejouer()

    1.6 Connecter la fin de partie

    Votre script ReactionDepot (ou équivalent) détecte déjà quand la sphère est déposée dans le support. Ajoutez-y une référence au GameManager et appelez TerminerJeu() au bon moment.

    [SerializeField] private GameManager gameManager;

// Dans la méthode déclenchée par selectEntered du socket :
gameManager.TerminerJeu();

    1.7 Tester

    • [ ] La scène démarre sur le menu
    • [ ] Pointer le bouton "Jouer" avec le contrôleur fait apparaître la ligne sur le Canvas
    • [ ] Appuyer sur la gâchette sur "Jouer" lance le jeu et affiche le HUD
    • [ ] Le timer s'incrémente
    • [ ] Déposer la sphère dans le support déclenche l'écran de fin avec un score
    • [ ] Le bouton "Rejouer" relance la scène

    Partie 2 — Adaptation au jeu de tri

    Vous maîtrisez maintenant les briques de base. Ouvrez votre scène TriSpatial et adaptez ce que vous venez de construire pour compléter votre jeu.

    Voici ce que vous devez réaliser :

    Ce qui reste identique

    • La configuration de l'Event System (XR UI Input Module)
    • La création et configuration des trois Canvas
    • La structure du GameManager (états, timer, score, boutons)

    Ce qui doit être adapté

    Le CanvasMenu doit apparaître devant le joueur dans votre environnement de tri — ajustez la position en conséquence.

    Le GameManager doit rester inactif au démarrage : les objets sont présents dans la scène, mais le tri ne peut pas commencer tant que le joueur n'a pas appuyé sur "Jouer". Réfléchissez à ce que ça implique pour les interactions grab.

    Indice : XRGrabInteractable peut être activé ou désactivé via enabled. Le GameManager peut désactiver les grabs au démarrage et les réactiver dans CommencerJeu().

    La fin de partie est déclenchée par le GestionnaireTri plutôt que par un seul socket. Modifiez GestionnaireTri pour qu'il appelle gameManager.TerminerJeu() au lieu d'afficher dans la console.

    Le score reflète le temps pour trier les trois objets — la formule de l'exemple convient, mais vous pouvez l'ajuster.

    Liste de vérification finale

    • [ ] La scène démarre sur le menu — les objets ne peuvent pas être grabbés
    • [ ] "Jouer" démarre le timer et active les interactions
    • [ ] Les trois objets déposés dans les bons réceptacles déclenchent l'écran de fin
    • [ ] Le score s'affiche correctement sur l'écran de fin
    • [ ] "Rejouer" relance la scène depuis le menu

    Défis supplémentaires

    Défi 1 — Réapparition des objets tombés

    Si un objet tombe hors de la salle (position Y inférieure à -1), remettez-le à sa position initiale.

    private Vector3 positionDepart;

void Start()
{
    positionDepart = transform.position;
}

void Update()
{
    if (transform.position.y < -1f)
    {
        transform.position = positionDepart;
        GetComponent<Rigidbody>().linearVelocity = Vector3.zero;
    }
}

    Défi 2 — Meilleur score persistant

    Sauvegardez le meilleur score entre les parties avec PlayerPrefs et affichez-le sur l'écran de fin.

    // Sauvegarder après calcul du score
PlayerPrefs.SetInt("MeilleurScore", score);
PlayerPrefs.Save();

// Lire pour affichage
int meilleurScore = PlayerPrefs.GetInt("MeilleurScore", 0);

    Défi 3 — Compte à rebours plutôt que chronomètre

    Remplacez le timer ascendant par un décompte de 60 secondes. Si le temps expire avant que le tri soit complété, affichez un écran "Raté" distinct.
