    Introduction à ARCore

    Qu'est-ce qu'ARCore?

    ARCore est la plateforme de réalité augmentée développée par Google pour les appareils Android. Elle permet aux développeurs de créer des expériences AR en utilisant les capteurs et caméras déjà présents sur les smartphones.

    Note : ARKit (Apple) fonctionne de manière très similaire. Les concepts appris avec ARCore sont transférables à ARKit.

    Design

    Les trois piliers d'ARCore

    ARCore repose sur trois capacités fondamentales.

    1. Tracking de mouvement (Motion Tracking)

    ARCore utilise une technique appelée SLAM (Simultaneous Localization and Mapping) pour comprendre où se trouve l'appareil dans l'espace.

    Comment ça fonctionne :

    • La caméra capture des images continuellement
    • ARCore identifie des points caractéristiques (feature points) dans l'image
    • En suivant ces points entre les images, ARCore calcule le déplacement de l'appareil
    • Résultat : ARCore sait où vous êtes et dans quelle direction vous regardez

    Analogie : Imaginez que vous fermez les yeux dans une pièce que vous connaissez. En touchant les meubles autour de vous, vous pouvez déduire votre position et vos mouvements. ARCore fait la même chose avec la caméra.

    Ce que ça permet :

    • Placer un objet virtuel et le voir rester au même endroit quand vous vous déplacez
    • Marcher autour d'un objet 3D pour le voir sous tous les angles
    • Créer une carte de l'environnement

    2. Détection de plans (Plane Detection)

    ARCore analyse l'environnement pour détecter des surfaces planes comme les sols, tables, ou murs.

    Comment ça fonctionne :

    • ARCore groupe les points caractéristiques qui sont à la même hauteur
    • Il forme des polygones qui représentent des surfaces
    • Ces surfaces sont continuellement mises à jour pour gagner en précision

    Types de plans détectés :

    Type Description Exemples
    Horizontal vers le haut Surface horizontale face vers le ciel Sol, table, bureau
    Horizontal vers le bas Surface horizontale face vers le sol Plafond
    Vertical Surface verticale Murs, portes

    Ce que ça permet :

    • Placer des objets de manière réaliste sur les surfaces
    • Savoir si une surface est un sol, un mur ou un plafond
    • Créer des interactions qui respectent la géométrie réelle

    3. Estimation de l'éclairage (Light Estimation)

    ARCore estime l'éclairage ambiant de l'environnement réel pour que les objets virtuels aient un rendu cohérent.

    Ce qui est estimé :

    • Intensité lumineuse : Est-ce lumineux ou sombre?
    • Température de couleur : La lumière est-elle chaude (jaune) ou froide (bleue)?
    • Direction principale : D'où vient la source de lumière?

    Ce que ça permet :

    • Objets virtuels qui s'intègrent naturellement
    • Ombres cohérentes avec l'environnement
    • Réflexions réalistes sur les objets virtuels

    Exemple concret :

    Si vous êtes dans une pièce éclairée par une lumière orange, vos objets virtuels auront aussi une teinte orange plutôt que d'être blanc pur.

    Concept clé : Les Ancres (Anchors)

    Une ancre est un point fixe dans l'espace réel auquel on attache des objets virtuels.

    Pourquoi les ancres sont essentielles?

    Sans ancre, ARCore doit recalculer constamment la position de chaque objet. Avec une ancre :

    • ✅ ARCore gère automatiquement la position de l'ancre
    • ✅ Les objets restent stables même quand vous bougez
    • ✅ La performance est optimisée

    Quand créer une ancre?

    • Placement d'objet : Quand l'utilisateur place un objet sur une surface
    • Persistance : Pour que l'objet reste là entre les sessions
    • Tracking précis : Pour des objets qui doivent rester parfaitement alignés

    Types d'ancres

    Type Description Usage
    Plane Anchor Ancré à un plan détecté Objets sur sol/table
    Feature Point Anchor Ancré à un point caractéristique Objets en l'air
    Cloud Anchor Ancré dans le cloud (partageable) Expériences multi-utilisateurs

    Système de coordonnées

    ARCore utilise un système de coordonnées différent :

    • Origine : Position de l'appareil au démarrage
    • Y+ : Vers le haut (gravité inversée)
    • Z+ : Direction où regarde la caméra au démarrage
    • X+ : Droite de la caméra

    Important : Les objets sont placés en mètres réels, pas en unités Unity arbitraires.

    AR Foundation : Le pont entre Unity et ARCore

    AR Foundation est une couche d'abstraction créée par Unity qui unifie ARCore (Android) et ARKit (iOS).

    Architecture

    Votre code Unity ↓ AR Foundation (API unifiée) ↓ ↓ ARCore ARKit (Android) (iOS)

    Avantages

    • ✅ Un seul code pour Android et iOS
    • ✅ API simplifiée par rapport à ARCore natif
    • ✅ Composants Unity intégrés
    • ✅ Simulation dans l'éditeur Unity

    Composants principaux d'AR Foundation

    Composant Rôle
    AR Session Gère la session AR (tracking, mise à jour)
    AR Session Origin Point de référence de la scène AR
    AR Camera Caméra spéciale pour afficher le monde réel
    AR Plane Manager Gère la détection de plans
    AR Raycast Manager Permet de faire des raycasts AR
    AR Anchor Manager Gère les ancres

    Différences clés : AR vs développement 3D classique

    Aspect 3D Classique AR avec ARCore
    Environnement Entièrement contrôlé Monde réel + virtuel
    Caméra Contrôle libre Position = position réelle de l'appareil
    Surfaces Créées manuellement Détectées automatiquement
    Lumière Configurée manuellement Estimée depuis l'environnement
    Échelle Arbitraire Mètres réels
    Input Clavier/souris Tactile uniquement

    Limites et défis d'ARCore

    Il est important de connaître les limites :

    Limitations techniques

    • ❌ Surfaces transparentes : Difficiles à détecter (verre, eau)
    • ❌ Surfaces uniformes : Peu de points caractéristiques (mur blanc)
    • ❌ Objets en mouvement : Peuvent perturber le tracking
    • ❌ Éclairage faible : Détection moins précise
    • ❌ Surfaces brillantes : Peuvent causer des reflets

    Bonnes pratiques

    • ✅ Demander à l'utilisateur de bien éclairer la pièce
    • ✅ Encourager le mouvement lent pour un meilleur tracking
    • ✅ Fournir du feedback visuel pendant la détection
    • ✅ Tester dans différents environnements
